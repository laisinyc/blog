### Frequently used Tags

1. single/multiple select: `input type=[radio|check]`
   - use `name` to group selection
   - use `value` as the actual option value
   - default select: `checked='checked'`

```html
<!-- single select -->
<input type='radio' name='single' value='abc'>
<input type='radio' name='single' value='123' checked='checked'>
<input type='radio' name='single' value='xxx'>

<!-- multiple select -->
<input type='checkbox' name='multi' value='abc'>
<input type='checkbox' name='multi' value='123'>
<input type='checkbox' name='multi' value='xxx'>
```

2. dropdown menu: `select name='xxx'`
   - wrap around `option` list
   - text between `option` are the value (if prop `value` no set)
   - `<option value='123'>abc</option>` if use `value` prop the submitted value will be from `value` prop

```html
<select name='province'>
  <option>beijing</option>
  <option>nanjing</option>
  <!-- use sh as acutal value -->
  <option value='sh'>shanghai</option>
</select>
```

## 01. 补充重要知识

1. content-box vs border-box

   - content-box: default value, will consider border and padding as part of width and height
   - boarder-box: more intuitive, it uses given width and height, border and padding as addition size.

2. transparency control (alpha)

   - take value between [0, 1]: rgba or hsla
   - 0: total transparent
   - 1: not transparent

   ```css
   .alpha {
     /* 一个完全透明的颜色，等同于 transparent */
     color: rgba(0, 0, 0, 0);
     /* 一个完全不透明的颜色，等同于rgb */
     color: rgba(0, 0, 0, 1);
     /* 一个半透明的颜色 */
     color: rgba(0, 0, 0, 0.5);
   }
   ```

3. percentage unit (15min)
   - relative unit, we need to have a **reference** to calculate actual size
   - non-positioned element reference **the parent element's content space**
     - 注意是 **内容区域**，不包括 `padding`, `border`, `margin`
   - positioned element reference it's first positioned parent element's **padding space** (measure from padding)
     - 注意这里是 `padding` + `内容`，即从 `padding` 开始计算
   - height: when use percentage height, the children element will look for it's parents height if parent height also depends on it's child height then the setting will be invalid!

| css prop | percentage relative to     | NOTE                                                                         |
| -------- | -------------------------- | ---------------------------------------------------------------------------- |
| width    | reference element's width  |                                                                              |
| height   | reference element's height | if reference element depends on referencer, the setting will not take effect |
| padding  | reference element's width  |                                                                              |
| border   | reference element's width  |                                                                              |
| margin   | reference element's width  |                                                                              |

4. min-width & max-width

   - min width: control minimum width of a page
   - max width: limit image size

5. z-index 的取值
   - 弹出菜单 -> 两位数
   - 固定菜单 -> 三位数
   - 总之，不能随便取一个数字，要事先约定好

## 03. 表单进阶-CSS

1. sudo class: hover, focus, disabled, checked
2. sudo element: before, after, placeholder
3. float height colapse: clear fix hack
4. inline(inline-block) vertical align fix
5. css selectors: ` `, `>`, `~`, `+`

```html
<!-- if an element contains only float element, it's height will be 0 -->
<!-- 'form-item' height will be 0, because 'left', 'right' are float element -->
<div class="form-item">
  <button class="left" disabled="disabled">Register</button>
  <button class="right">Reset</button>
</div>
```

```html
<!-- clear fix hack 1 -->
<!-- add a block element after float elements -->
<div class="form-item">
  <button class="left" disabled="disabled">Register</button>
  <button class="right">Reset</button>
  <span style="
    display: block;
    /* The element is pushed below both left and right floated elements */;
    clear: both;
  "></span>
</div>
```

```css
/* clear fix hack 2 */
/* sudo element */
.clearfix::after {
  content: '';
  display: block;
  clear: both;
}
```

3. `checkbox` and `span` are not aligned vertically, how to fix?

```html
<!-- because both checkbox: inline-block; span: inline are inline elements -->
<div class="form-item policy">
  <label>
    <input type="checkbox" />
    <span>同意台湾是中国领土不可分割的一部分</span>
  </label>
</div>
```

```css
/* use vertical align to adjust any element as needed */
.policy span {
  vertical-align: 2px;
}
```

4. css ` `, `>`, `~`, `+` selector

- ` `: descendant selector; children, grandchildren, etc.
- `>`: child selector; direct children
- `~`: sibling selector; select all siblings
- `+`: select the first sibling

## 06. 属性值的计算过程

- CSS 属性值的计算过程: 无属性 -> 计算(见下面四部) -> 每个元素都有值
  1. 确定声明值：参考样式表中没有冲突的声明，作为 css 属性值
  2. 层叠冲突：对样式表有冲突的声明使用层叠规则，确定 css 属性值
     - 层叠规则：
       1. 比较重要性；作者样式表替代浏览器默认样式
       2. 比较特殊性（权重计算规则）
       3. 比较源次序（后出现的替换前面出现的）
  3. 使用继承：对仍然没有值的属性，如果可以继承，则继承父元素的值
     - 文字相关可以继承
  4. 使用默认值：对任然没有值的属性，使用默认值
- 为什么给 a 元素的父元素设置颜色，无法影响 a 元素
  - `a` 元素自带属性，需要手动设置去 override

## 07. 拓展知识

- sudo class selectors:
  - `:link` 选中未访问的链接
  - `:visited` 选中已访问的链接
  - `:hover` 选中鼠标移入的元素
  - `:active` 选中鼠标按下的元素
  - `:focus` 选中聚焦的**表单**元素
  - `:disabled` 选中被禁用的**表单**元素
  - `:checked` 选中被选中的**表单**元素
  - `:first-child`
  - `:last-child`
  - `:nth-child(an+b)` 选中第 `an+b` 个元素（a,b 为常量，n 从 0 开始递增）
    - 选中奇数元素：`:nth-child(2n + 1)`
  - `:first-of-type` 选中第一个指定类型的子元素
  - `:last-of-type`
  - `:nth-of-type(an+b)`
- `contenteditable`属性：`<div contenteditable>...</div>` 可以用来实现富文本框（已有现成插件，基本不再使用）
- `table`元素：css3 普及之后很少使用（渲染速度慢前台很少使用）
